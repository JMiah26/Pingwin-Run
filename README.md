# PingWin Run

PingWin Run is a 3D platform game made in Unity.

The game is about moving through obstacle courses, collecting coins, avoiding danger, and reaching the end of the level.

## Project Overview

For this project, I worked on a 3D parkour style game where the player has to get from one point to another without falling or getting caught. The game is not really about speed. It is more about movement, timing, and getting through the level safely.

## Gameplay

The player runs, jumps, collects coins, avoids enemies, and uses keys to open doors. There are also checkpoints in the level to save progress. If the player falls off the map or hits danger, they respawn and try again.

## Main Features

- 3D player movement
- Third person camera
- Coin pickups
- Simple score system
- Enemy follow system
- Coloured keys and doors
- Checkpoints
- Main menu and pause menu

## What I Worked On

For this project, I worked on the movement system, camera controls, pickups, health and respawn system, UI, menus, enemies, keys, doors, checkpoints, and character ideas. Most of the project was about building game mechanics and making them work together properly.

## Movement and Camera

A big part of the game was getting the movement to feel right. I used a character controller to handle player movement, jump force, gravity, and direction. I also worked on a third person camera that follows the player and moves with the mouse so the player can look around more easily.

## Pickups and Health

The coin system lets the player collect coins and increase their score. For health, I kept things simple. Instead of using a health bar, the game resets the player if they fall into a kill box or make a mistake. This made the game feel more challenging.

## Enemies and Progression

The enemies follow the player when they get close, then stop chasing if the player moves too far away. I also added a key and door system where the player has to collect the right coloured key to open the matching door. Checkpoints were added as well so the player can respawn at the latest point they reached.

## UI and Menus

The project includes a coin counter, a main menu, and a pause menu. The UI updates when coins are collected, and the menu system lets the player start, pause, resume, or quit the game.

## Tools Used

- Unity
- C#
- Autodesk Maya

## What I Learned

This project helped me improve my understanding of player movement, camera systems, pickups, UI design, enemy behaviour, checkpoints, and general gameplay programming in Unity. It also helped me see how small systems need to work together to make a full game feel complete.

## Challenges

One of the main challenges was making the movement and camera feel smooth and easy to control. Another was getting different systems such as enemies, keys, doors, checkpoints, and menus to all work properly in the same project.

## Future Improvements

If I continued this project, I would improve the character camera control, refine the levels, make the movement feel even better, and add more polish to the UI and gameplay. I would also spend more time improving the visual side of the game so it feels more finished overall.

## Summary

PingWin Run is a simple 3D game project that helped me build my skills in Unity. It gave me practice with player movement, cameras, pickups, enemies, UI, and level progression. Overall, it was a good way to learn how different game systems come together in one playable project.
